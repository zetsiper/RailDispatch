# RailDispatch 🚂📊

Интелигентно уеб приложение за 24-часово оперативно планиране, управление и отчитане на локомотивни бригади в товарния железопътен транспорт в България.

Системата е проектирана да работи в среда на висока неопределеност (нерегулярни влакове, локомотивни повреди и недостиг на персонал), като осигурява строго и точно отчитане на оперативното състояние преди преминаване към автоматични оптимизационни предложения.

---

## 🌟 Ключови характеристики (Key Features)

- **24-часов графичен Gantt диаграма:** Динамична времева линия, изобразяваща състоянието на всеки машинист (Зелено за дежурство, Сиво за задължителна почивка, Червено за болничен/отпуск и Свободно време).
- **Интерактивен симулатор на времето:** Плъзгач в горната част на екрана позволява промяна на „диспечерското време“, като незабавно преизчислява състоянията, наличността и статистиката за деня.
- **Двигател за бизнес правила (Logic Engine):** Автоматична валидация по метода на „Светофара“ преди всяко одобрение на смяна.
- **Динамичен редактор на влакове:** Секция за бърза промяна, добавяне или изтриване на редовни влакове/задачи (Влак ID, маршрут, изисквана серия локомотив и ETA времетраене).
- **Управление на събития по време на път:** Възможност за регистриране на закъснения, преназначаване или отмяна на влак по причини (повреда, проблеми по инфраструктурата, липса на състав).

## 🚦 Бизнес правила и ограничения (Logic Engine)

1. **Задължителна почивка (Твърдо ограничение):**
   - Минималната почивка между две дежурства е точно **16 часа**.
   - `Възможно_Следващо_Явяване = Освобождаване_Реално + 16 часа`.
   - Системата блокира всяко планирано явяване преди този срок и показва червена светлина.
2. **Правоспособност за локомотив (Твърдо ограничение):**
   - Машинистът трябва да притежава правоспособност за конкретния серия локомотив: **40 (вкл. 42/43/44/45), 46, 07, 51 и 55**.
   - Несъвместимост светва в червено и блокира назначението.
3. **Ротация на влаковете (Меко ограничение):**
   - Проверява последните 5 дежурства с цел избягване на последователно каране на един и същ влак от един машинист. Предупреждава със сигнал `⚠️ Повторение`.
4. **Баланс на часовете (Акумулатори):**
   - Изработеното време при приключване на всяка смяна се добавя към месечния лимит (~160ч) и тримесечния норматив за извънреден труд.
5. **Отсъствия и Отпуски (Твърдо ограничение):**
   - Персоналът в платен отпуск, болничен или лични неприсъствени часове е напълно скрит от списъците за оперативно планиране.

## 💻 Технологичен стек (Tech Stack)

Приложението е разработено по стандарти за максимална производителност и превъзходен UX без тежки зависимости:
- **Core:** HTML5, Vanilla JavaScript (ES6)
- **Styling:** Custom Vanilla CSS3 (Glassmorphism ефект, адаптивен за всякакви десктоп екрани, плавни анимации)
- **State & Persistence:** Локална персистентност чрез автоматично синхронизиране с `localStorage`.
- **Dev Server:** [Vite](https://vitejs.dev/) за бързо локално развитие и тестване.

## 🚀 Бърз старт (Getting Started)

### Начин 1: Директно отваряне
Тъй като приложението разчита на чисти уеб технологии, можете просто да отворите файла `index.html` директно във Вашия браузър (Chrome, Firefox, Edge, Safari).

### Начин 2: Локален сървър за разработка (Vite)
Ако имате инсталиран [Node.js](https://nodejs.org/):

1. Клонирайте хранилището.
2. Стартирайте инсталирането на Vite:
   ```bash
   npm install
   ```
3. Стартирайте сървъра за разработка:
   ```bash
   npm run dev
   ```
4. Отворете браузъра на адрес `http://localhost:3000`.
